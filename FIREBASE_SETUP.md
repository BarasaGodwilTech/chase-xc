# Firebase Database Setup for Chase x Records

## Overview
This document describes the Firebase Firestore database structure for the Chase x Records music platform.

## Collections

### 1. `artists` Collection
Stores artist information.

**Document Structure:**
```javascript
{
  id: string,           // Auto-generated by Firestore
  name: string,         // Artist name (required)
  genre: string,        // Primary genre (e.g., "Afro-Pop")
  bio: string,          // Artist biography
  image: string,        // URL to artist image (Firebase Storage)
  status: string,       // "active" or "inactive"
  createdAt: timestamp, // Server timestamp
  updatedAt: timestamp  // Server timestamp
}
```

**Example:**
```javascript
{
  name: "Sarah Miles",
  genre: "Afro-Pop",
  bio: "Soulful vocalist known for powerful performances",
  image: "https://firebasestorage.googleapis.com/...",
  status: "active",
  createdAt: Timestamp,
  updatedAt: Timestamp
}
```

### 2. `tracks` Collection
Stores music track information.

**Document Structure:**
```javascript
{
  id: string,              // Auto-generated by Firestore
  title: string,           // Track title (required)
  artist: string,          // Artist ID (reference to artists collection)
  artistName: string,      // Artist name (denormalized for easier access)
  genre: string,           // Track genre (required)
  duration: string,        // Duration in "MM:SS" format
  streams: number,         // Play count (default: 0)
  likes: number,           // Like count (default: 0)
  downloads: number,       // Download count (default: 0)
  status: string,          // "published" or "draft"
  artwork: string,         // URL to track artwork (Firebase Storage)
  audioUrl: string,        // URL to audio file (Firebase Storage)
  releaseDate: string,     // Release date in "YYYY-MM-DD" format
  description: string,     // Track description
  platformLinks: {         // Links to streaming platforms
    spotify: string,
    appleMusic: string,
    youtube: string,
    soundcloud: string
  },
  createdAt: timestamp,    // Server timestamp
  updatedAt: timestamp     // Server timestamp
}
```

**Example:**
```javascript
{
  title: "Sunset Dreams",
  artist: "artist123",
  artistName: "Sarah Miles",
  genre: "Afro-Pop",
  duration: "3:45",
  streams: 15200,
  likes: 2400,
  downloads: 1800,
  status: "published",
  artwork: "https://firebasestorage.googleapis.com/...",
  audioUrl: "https://firebasestorage.googleapis.com/...",
  releaseDate: "2024-12-15",
  description: "A soulful afro-pop track",
  platformLinks: {
    spotify: "https://open.spotify.com/...",
    appleMusic: "https://music.apple.com/...",
    youtube: "https://youtube.com/...",
    soundcloud: "https://soundcloud.com/..."
  },
  createdAt: Timestamp,
  updatedAt: Timestamp
}
```

## Firebase Storage Structure

### Storage Paths:
- **Artist Images:** `artists/{timestamp}.{ext}`
- **Audio Files:** `audio/{artistId}/{timestamp}.{ext}`
- **Track Artwork:** `artwork/{artistId}/{timestamp}.{ext}`

## Firestore Rules (Auto-Creation Enabled)

These rules allow the database to be created automatically on first upload:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Artists collection - auto-created on first write
    match /artists/{artistId} {
      allow read: if true;
      allow write: if request.auth != null || request.resource.data != null;
    }
    
    // Tracks collection - auto-created on first write
    match /tracks/{trackId} {
      allow read: if true;
      allow write: if request.auth != null || request.resource.data != null;
    }
  }
}
```

**For Development (Open Access):**
If you want to test without authentication first:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}
```

**Note:** Switch to authenticated rules before production deployment.

## Storage Rules (Auto-Creation Enabled)

These rules allow storage folders to be created automatically on first upload:

```javascript
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /artists/{allPaths=**} {
      allow read: if true;
      allow write: if request.auth != null || request.resource != null;
    }
    match /audio/{allPaths=**} {
      allow read: if true;
      allow write: if request.auth != null || request.resource != null;
    }
    match /artwork/{allPaths=**} {
      allow read: if true;
      allow write: if request.auth != null || request.resource != null;
    }
  }
}
```

**For Development (Open Access):**
If you want to test without authentication first:

```javascript
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /{allPaths=**} {
      allow read, write: if true;
    }
  }
}
```

**Note:** Switch to authenticated rules before production deployment.

## Admin Panel Usage

### Adding Artists
1. Go to Admin Panel → Artists section
2. Click "Add Artist" button
3. Fill in artist details (name, genre, bio)
4. Upload artist image (optional)
5. Click "Save Artist"

### Uploading Tracks
1. Go to Admin Panel → Music Management section
2. Select "Upload" method
3. Fill in track details (title, genre, release date, duration)
4. Select artist from dropdown (or add new artist)
5. Upload audio file (required)
6. Upload artwork (optional)
7. Click "Upload Track"

The track will be:
- Uploaded to Firebase Storage
- Added to Firestore `tracks` collection
- Automatically available on the music page
