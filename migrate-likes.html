Migrate Track Likes

What this does:

  • Fetches all users from Firestore
  • Counts how many users liked each track (from their favorites)
  • Updates each track's likes field with the correct count

Why run this?

  • Before this implementation, likes were only stored in user favorites
  • Now likes are also counted in the track's likes field
  • This migration backfills the counts for tracks liked before the change

Note:

  • This is a one-time migration script
  • After running, new likes will be automatically counted
  • Make sure you have proper Firebase permissions
